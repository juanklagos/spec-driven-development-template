# Contrato funcional - Lista de tareas personal

## Reglas de comportamiento

1. Si el texto de la tarea está vacío, no se debe crear la tarea.
2. Cada tarea creada debe verse inmediatamente en la lista de pendientes.
3. Al marcar una tarea como completada, debe cambiar de lista (de pendientes a completadas).
4. Al eliminar una tarea, debe desaparecer de la interfaz.

## Reglas de experiencia de usuario

1. Debe existir un botón visible para agregar tarea.
2. Debe existir un control visible para completar tarea.
3. Debe existir un control visible para eliminar tarea.
