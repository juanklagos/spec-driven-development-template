# Reglas para especificaciones

Cada especificación debe crearse con número y nombre.

Ejemplo:

- `001-inicio`
- `002-nueva-funcionalidad`

Cada especificación debe incluir, sin excepción:

- `spec.md`
- `plan.md`
- `tasks.md`
- `research.md`
- `contracts/` cuando sea necesario
