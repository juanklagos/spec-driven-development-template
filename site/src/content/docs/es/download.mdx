**SDD Desk** es el builder visual en su propia ventana. Trae su propio Node adentro, hospeda el
servidor SDD él mismo, e incluye el endpoint MCP al que se conectan tus agentes.

## Antes de descargar: cuál te conviene

Hay dos formas de usar el mismo builder, y la mejor depende de una sola cosa — si te molesta
autorizar una app la primera vez que la abres.

<CardGrid>
  <Card title="El builder en el navegador" icon="laptop">
    **Sin ningún aviso, y sin descargar nada.** Un comando y se abre:

    ```bash
    npx @juanklagos/sdd-mcp@latest --http
    ```

    Mismo lienzo, mismo servidor, mismo endpoint MCP. **Necesita Node instalado.**
  </Card>
  <Card title="SDD Desk" icon="rocket">
    **Ventana propia, ícono propio, y trae Node adentro** — no hay que instalar nada antes.

    A cambio: macOS y Windows **te pedirán autorizarla la primera vez**, con un aviso que suena
    alarmante. Son tres clics, una sola vez, y aquí abajo está el paso a paso.
  </Card>
</CardGrid>

<Aside type="caution" title="Si nunca has autorizado una app a mano, empieza por el navegador">
La app de escritorio no está firmada con un certificado de Apple ni de Microsoft, así que tu
sistema mostrará un aviso al abrirla y te sugerirá mover el archivo a la papelera. No es que la
descarga esté rota — [aquí está qué significa](#por-qué-aparece-ese-aviso) — pero si esa
frase te pone nervioso, el builder en el navegador hace exactamente lo mismo sin pedirte nada.
</Aside>

## Por qué aparece ese aviso

Firmar un programa y publicarlo con licencia abierta son dos cosas distintas, y es fácil
confundirlas.

**La licencia dice qué puedes hacer con el código.** SDD Desk es MIT: puedes usarlo, leerlo,
modificarlo, redistribuirlo y venderlo. El código está entero en el repositorio, incluido el
que construye estas descargas.

**La firma dice quién armó el archivo que descargaste.** macOS y Windows no leen licencias:
comprueban si un certificado de pago acredita la identidad de quien firmó el binario. Un `.dmg`
MIT sin firmar y uno propietario sin firmar reciben exactamente el mismo aviso.

Así que el aviso **no dice** que la app esté rota, ni que sea peligrosa, ni que tu sistema haya
detectado algo. Dice, literalmente, que el sistema no puede verificar quién la publicó. Que sea
correcto sortearlo depende de si confías en la fuente, y para eso está el código a la vista.

**Y no está firmada por decisión, no por olvido.** Apple cobra 99 USD al año por el certificado
que quita el aviso, sin excepción para proyectos abiertos. El proyecto lo mantiene una sola
persona y prefirió no atar la herramienta a una cuota anual: el builder en el navegador ya
cubre sin fricción a quien no quiera lidiar con esto, y el código está ahí para compilarlo.

Windows sí tiene una vía gratuita para licencias aprobadas por OSI, que MIT cumple, y está
pendiente de solicitar. Linux no pide firma de nada.

## Abrirla la primera vez

<Tabs syncKey="os">
  <TabItem label="macOS" icon="apple">
    Abre el `.dmg` y arrastra **SDD Desk** a Aplicaciones. Al abrirla por primera vez, macOS
    dirá que *"no pudo verificar que no contenga software malicioso"*: eso es lo que dice de
    cualquier app sin notarizar. Solo hay que autorizarla una vez.

    **En macOS 15 (Sequoia) o posterior** — incluido macOS 26:

    1. Intenta abrirla con doble clic. Saldrá el aviso; pulsa **Listo**, no "Mover al basurero".
    2. Ve a **Ajustes del Sistema → Privacidad y Seguridad** y baja hasta *Seguridad*.
    3. Verás que SDD Desk fue bloqueada: pulsa **Abrir de todos modos** y confirma con tu
       contraseña.

    Apple [quitó el atajo de clic derecho → Abrir](https://appleinsider.com/articles/24/08/06/apple-removes-control-click-option-for-skipping-gatekeeper-in-macos-sequoia)
    en Sequoia, así que en estas versiones ese camino ya no existe.

    **En macOS 14 (Sonoma) o anterior:** clic derecho sobre la app → **Abrir**, y confirmar.

    En cualquier versión, una sola orden hace lo mismo — úsala solo sobre descargas en las que
    confíes, porque quita la comprobación que macOS acaba de hacer por ti:

    ```bash
    xattr -dr com.apple.quarantine "/Applications/SDD Desk.app"
    ```

    <Aside type="danger" title="Si dice «está dañado y no puede abrirse»">
    Eso **no** es el aviso normal, y no significa que la descarga viniera mal. Fue un fallo de
    empaquetado en la versión 0.1.1: el bundle salió sin un sello de firma válido, y Apple
    Silicon rechaza eso de plano en vez de pedir confirmación. Está arreglado desde la 0.1.2 —
    vuelve a descargar y no volverá a pasar. Si prefieres rescatar la copia que ya bajaste:

    ```bash
    xattr -dr com.apple.quarantine "/Applications/SDD Desk.app"
    codesign --force --deep --sign - "/Applications/SDD Desk.app"
    ```
    </Aside>

  </TabItem>
  <TabItem label="Windows" icon="seti:windows">
    SmartScreen mostrará "Windows protegió tu PC". Haz clic en **Más información** y luego en
    **Ejecutar de todas formas**. El aviso sale porque el instalador no está firmado, y seguirá

## Compilarla tú mismo

La app es MIT, como todo el proyecto. Un binario que compilas en tu máquina no viene de
internet, así que ningún sistema lo pone en cuarentena y no verás ningún aviso:

```bash
git clone https://github.com/juanklagos/spec-driven-development-template.git
cd spec-driven-development-template && npm ci && npm run build
cd builder && npm ci && npm run build && cd ../desk
npm install && npm run dist
```

El resultado queda en `desk/release/`. Necesita Node solo para compilar; la app resultante no.

## Qué pasa cuando la abres

1. Te pide la carpeta de tu proyecto — una que tenga `specs/` o el sidecar `spec/`. ¿No tienes
   ninguna? Créala con `npx @juanklagos/create-sdd-project@latest .`
2. Se abre el lienzo sobre ese proyecto. Tus archivos en disco siguen siendo la fuente de
   verdad; la app no los copia a ningún lado.
3. Recuerda la carpeta. **Archivo → Abrir proyecto…** cambia a otra sin reiniciar.

## Conectar un agente

Mientras está abierta, la app **es** el servidor MCP de tu proyecto. En
**Archivo → Conectar un agente…** ves la URL y un fragmento listo para copiar:

```json
{ "mcpServers": { "sdd": { "url": "http://127.0.0.1:3334/mcp" } } }
```

El puerto es el que la app realmente tomó — se corre al siguiente libre si 3334 está ocupado,
y por eso el diálogo te enseña el número real en vez de que lo supongas.

Todo lo que tu agente escriba por ese endpoint aparece en la ventana al instante, sin recargar.

## Qué no hace

Corre entera en tu máquina. No hay cuenta, ni sincronización, ni un servidor nuestro de por
medio — la app abre un puerto en `127.0.0.1` que nada fuera de tu computadora alcanza.

Tampoco reemplaza la ruta de terminal. `npx @juanklagos/sdd-mcp@latest --http` sigue abriendo
el mismo builder en tu navegador, y sigue funcionando igual que antes.
