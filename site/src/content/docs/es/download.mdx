**SDD Desk** es el builder visual en su propia ventana. Trae su propio Node adentro, así que no
hay que instalar nada antes, y hospeda el servidor SDD él mismo — incluido el endpoint MCP al
que se conectan tus agentes.

<Aside type="caution" title="Todavía no están firmadas">
Los certificados de firma de Apple y Windows cuestan dinero que el proyecto aún no ha gastado,
así que tu sistema operativo **te va a advertir** la primera vez que abras la app. La
advertencia es cierta: significa que la descarga no está verificada, no que sea segura. Si
prefieres no sortearla, [usa el builder en el navegador](../guides/51-sdd-builder-visual-guide/)
— las mismas funciones, un comando, sin advertencia.
</Aside>

## Abrirla la primera vez

<Tabs syncKey="os">
  <TabItem label="macOS" icon="apple">
    macOS se niega a abrir apps sin firmar descargadas de internet. Abre el `.dmg`, arrastra
    **SDD Desk** a Aplicaciones, y después **haz clic derecho sobre la app y elige Abrir** — ese
    diálogo trae un botón de Abrir que el del doble clic no tiene.

    Si macOS dice que la app "está dañada", es la marca de cuarentena y no un daño real:

    ```bash
    xattr -dr com.apple.quarantine "/Applications/SDD Desk.app"
    ```

    Ejecútalo solo sobre una descarga en la que confíes. Quita la comprobación que macOS
    acaba de hacer por ti.
  </TabItem>
  <TabItem label="Windows" icon="seti:windows">
    SmartScreen mostrará "Windows protegió tu PC". Haz clic en **Más información** y luego en
    **Ejecutar de todas formas**. El aviso sale porque el instalador no está firmado, y seguirá
    saliendo hasta que el proyecto tenga certificado.
  </TabItem>
  <TabItem label="Linux" icon="linux">
    Dale permiso de ejecución y ábrela:

    ```bash
    chmod +x SDD-Desk-linux.AppImage
    ./SDD-Desk-linux.AppImage
    ```

    Al lado se publica un `.deb` para Debian y Ubuntu. No hay Flatpak: Flathub prohíbe el
    código asistido por IA desde el 2026-05-29.
  </TabItem>
</Tabs>

## Dos caminos sin ningún aviso

La app es MIT, como todo el proyecto. Si prefieres no lidiar con Gatekeeper ni SmartScreen,
hay dos formas de usar exactamente lo mismo sin que ningún sistema te advierta nada.

**El builder en el navegador.** Un comando, sin descargar nada:

```bash
npx @juanklagos/sdd-mcp@latest --http
```

Es el mismo lienzo, el mismo servidor y el mismo endpoint MCP. Lo único que no te da es la
ventana propia y el ícono en el Dock — y necesita Node instalado, que es justamente lo que la
app de escritorio viene a evitar.

**Compilar la app tú mismo.** Un binario que compilas en tu máquina no viene de internet, así
que ningún sistema lo pone en cuarentena:

```bash
git clone https://github.com/juanklagos/spec-driven-development-template.git
cd spec-driven-development-template && npm ci && npm run build
cd builder && npm ci && npm run build && cd ../desk
npm install && npm run dist
```

El resultado queda en `desk/release/`. Necesita Node solo para compilar; la app resultante no.

## Qué pasa cuando la abres

1. Te pide la carpeta de tu proyecto — una que tenga `specs/` o el sidecar `spec/`. ¿No tienes
   ninguna? Créala con `npx @juanklagos/create-sdd-project@latest .`
2. Se abre el lienzo sobre ese proyecto. Tus archivos en disco siguen siendo la fuente de
   verdad; la app no los copia a ningún lado.
3. Recuerda la carpeta. **Archivo → Abrir proyecto…** cambia a otra sin reiniciar.

## Conectar un agente

Mientras está abierta, la app **es** el servidor MCP de tu proyecto. En
**Archivo → Conectar un agente…** ves la URL y un fragmento listo para copiar:

```json
{ "mcpServers": { "sdd": { "url": "http://127.0.0.1:3334/mcp" } } }
```

El puerto es el que la app realmente tomó — se corre al siguiente libre si 3334 está ocupado,
y por eso el diálogo te enseña el número real en vez de que lo supongas.

Todo lo que tu agente escriba por ese endpoint aparece en la ventana al instante, sin recargar.

## Qué no hace

Corre entera en tu máquina. No hay cuenta, ni sincronización, ni un servidor nuestro de por
medio — la app abre un puerto en `127.0.0.1` que nada fuera de tu computadora alcanza.

Tampoco reemplaza la ruta de terminal. `npx @juanklagos/sdd-mcp@latest --http` sigue abriendo
el mismo builder en tu navegador, y sigue funcionando igual que antes.
