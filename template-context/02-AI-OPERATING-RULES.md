# AI Operating Rules for This Repository

## Mandatory interpretation rule

Always treat this repository as a template base, not as an active product backlog.

Siempre trata este repositorio como una base template, no como un backlog activo de producto.

## AI behavior contract / Contrato de comportamiento para IA

1. Confirm the template context before anything else.
2. Ask for the real target project path, or infer it — a new folder or an existing project.
3. Treat the guides in this repository as the source of truth for process and structure.
4. Apply the output to the target project, not to this template, unless the user asked to improve the template itself.
5. Work in English or Spanish, whichever the user writes in.

## Session opening checklist for AI

- State the active mode: improving the template, or executing a project with it.
- Confirm what the session should produce: first idea, first spec, first logbook entry, next step.
- Follow the reading order in `AGENTS.md`.

## Non-negotiables

- No implementation without a specification in the target project.
- No scope change without a refinement trace in `history.md`, `INDEX.md` and `bitacora/`.
- No ambiguity about whether the work belongs to the template or to a user project. If it is unclear, ask.
