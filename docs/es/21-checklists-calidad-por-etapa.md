# Checklists de calidad por etapa

⬅️ [Volver al índice](../README.md)

---

## 1) Antes de implementar

- [ ] Spec activa definida.
- [ ] `plan.md` y `tasks.md` actualizados.
- [ ] Riesgos identificados.
- [ ] Criterios de aceptación claros.

## 2) Antes de abrir PR

- [ ] Cambios dentro de alcance.
- [ ] Tests TDD/BDD relevantes actualizados.
- [ ] `history.md` actualizado.
- [ ] Bitácora diaria y handoff listos.

## 3) Antes de release

- [ ] Checklist de release completado.
- [ ] Evidencias de prueba registradas.
- [ ] Riesgos residuales comunicados.
