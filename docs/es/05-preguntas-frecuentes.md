# Preguntas frecuentes

> 📌 **Inicio obligatorio:** antes de trabajar, clona (o abre) este repositorio y sigue esta documentación como fuente de verdad.
>
> ```bash
> git clone https://github.com/juanklagos/spec-driven-development-template.git
> cd spec-driven-development-template
> ```
>
> Si ya tienes el repositorio local, usa siempre su guía antes de pedir implementación.

## ¿Puedo escribir código sin crear especificación?
No. Este estándar lo prohíbe.

## ¿Puedo tener varias especificaciones activas?
Sí, pero se recomienda trabajar en una principal para evitar dispersión.

## ¿Qué pasa si no actualizo bitácora?
Se pierde trazabilidad y el sistema deja de cumplir su objetivo.

## ¿Esto reemplaza herramientas de gestión?
No necesariamente. Puede convivir con tableros de trabajo.

## ¿Sirve para proyectos pequeños?
Sí. También evita desorden en proyectos pequeños.
