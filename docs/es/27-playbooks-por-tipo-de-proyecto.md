# Playbooks por tipo de proyecto

⬅️ [Volver al índice](../README.md)

---

Estos playbooks son opcionales. El template funciona igual si no los usas.

## Packs disponibles

- `playbooks/saas/`
- `playbooks/ecommerce/`
- `playbooks/mobile-app/`
- `playbooks/backend-api/`

## Cuándo usar cada pack

| Pack | Usar cuando |
|---|---|
| SaaS | Producto multi-tenant con suscripciones o panel |
| E-commerce | Flujo de catálogo, carrito, checkout y pagos |
| Mobile app | Navegación móvil, offline, sincronización |
| Backend API | Contratos de API, seguridad y observabilidad |

## Prompt para activar pack

```text
Usa este template y el playbook [PACK].
Propón primera idea y partición inicial de specs según ese tipo de proyecto.
```
