# Quality checklists by stage

---

## 1) Before implementation

- [ ] Active spec is defined.
- [ ] `plan.md` and `tasks.md` are up to date.
- [ ] Risks are identified.
- [ ] Acceptance criteria are clear.

## 2) Before opening PR

- [ ] Changes are inside scope.
- [ ] Relevant TDD/BDD tests are updated.
- [ ] `history.md` is updated.
- [ ] Daily log and handoff are ready.

## 3) Before release

- [ ] Release checklist completed.
- [ ] Test evidence recorded.
- [ ] Residual risks communicated.
