## 🗣️ Friendly prompt (copy/paste)

Use this when you are not technical and want the AI to do setup + guidance end-to-end:

```text
Using https://github.com/juanklagos/spec-driven-development-template, create everything needed to carry out my project end-to-end.
My project is: [describe your project in plain language].

If my project is new, initialize it with this template and GitHub Spec Kit.
If my project already exists, adapt it to idea/specs/bitacora without breaking current behavior.
Guide me step by step for my level (beginner/intermediate/advanced), using simple language.
Do not skip specification, plan, tasks, refinement trace, logbook, and validation.
```

## Can I code without a specification?
No. This standard does not allow it.

## Can I have multiple active specifications?
Yes, but one primary active specification is recommended.

## What happens if I skip logs?
Traceability is lost and the process quality drops.

## Does this replace project management tools?
Not necessarily. It can work together with boards and issue trackers.

## Is this useful for small projects?
Yes. It helps prevent disorder in small projects too.
