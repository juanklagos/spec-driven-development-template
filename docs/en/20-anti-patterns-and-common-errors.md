## 🚫 What not to do

| Anti-pattern | Problem | Fix |
|---|---|---|
| Implement without an active spec | Directionless changes | Define or activate a spec first |
| Change scope without history | Lost context | Record change in `history.md` |
| End session without handoff | Broken continuity | Create `bitacora/handoffs/` entry |
| Mix multiple goals in one spec | Priority chaos | Split into numbered specs |

## Warning signals

- `tasks.md` is outdated.
- `specs/INDEX.md` does not reflect reality.
- Logbook has no recent updates.

## Simple rule

If it is not documented, it is not aligned.
