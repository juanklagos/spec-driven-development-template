# SDD Consent / Consentimiento SDD

Consent is recorded PER SPEC. One approved spec needs one `[spec:<id>]` entry.
El consentimiento se registra POR SPEC. Cada spec aprobada necesita su entrada `[spec:<id>]`.

EN:
- Before execution/implementation starts for an approved spec, record explicit user consent.
- Command:
  `./scripts/confirm-user-consent.sh --spec NNN-slug "User approved implementation for spec NNN-slug"`

ES:
- Antes de iniciar ejecución/implementación sobre una spec aprobada, registra consentimiento explícito del usuario.
- Comando:
  `./scripts/confirm-user-consent.sh --spec NNN-slug "Usuario aprobó implementar la spec NNN-slug"`

Required log file / Archivo obligatorio:
- `.sdd/user-consent.log`

Entry format / Formato de entrada:
- `[<timestamp>] [spec:<NNN-slug>] <summary>`
- An entry with no `[spec:...]` marker is a legacy, workspace-wide record: the gate
  still accepts it, with a warning naming the migration command.
- Una entrada sin marcador `[spec:...]` es un registro heredado para todo el workspace:
  la compuerta la acepta, con una advertencia que indica el comando de migración.
